    Configuration

    ansede.json

    Create an ansede.json in your project root to configure the scanner:

    {
  "exclude_paths": [
    "tests/fixtures",
    "legacy_code",
    "__pycache__",
    "node_modules",
    ".git"
  ],
  "disable_rules": [
    "PY-013",
    "CWE-862"
  ],
  "custom_sources": [
    "get_untrusted_user_input",
    "request.headers.get"
  ],
  "custom_sinks": {
    "my_vulnerable_db_execute": {
      "cwe": "CWE-89",
      "title": "Custom SQL Injection sink",
      "severity": "critical"
    }
  }
}

    Fields

    Field Type Description
    exclude_paths string[] Path patterns to skip during scanning
    disable_rules string[] Rule IDs or CWEs to suppress entirely
    custom_sources string[] Function names treated as taint sources
    custom_sinks object Function names treated as taint sinks with CWE mapping
    custom_rules_file string Path to a YAML/JSON file with custom rules

    Initialize

    ansede-static --init

    Creates a starter ansede.json with sensible defaults.

    .ansedeignore

    A .ansedeignore file in the workspace root uses gitignore-compatible syntax:

    # Skip autogenerated files
gen/
*.generated.py

# Skip specific directories
legacy/
vendor/

    CLI reference

    Run ansede-static --help for the full CLI reference. Key flags:

    Flag Default Description
    --format text Output format: text, json, sarif, html, ciso
    --output stdout Write output to a file
    --fail-on high Severity gate for CI exit codes
    --batch off Batch mode with shared cache + parallel workers
    --incremental off Git diff mode — scan only changed files
    --baseline none Compare against a baseline JSON report
    --js-backend auto JS/TS engine: auto, classic, structural
    --engine auto Analysis engine: v1, v2, auto
    --audit off Post-scan verdict classification
    --min-confidence 0.0 Minimum confidence threshold (0.0–1.0)
    --workers cpu_count Parallel workers (implies --parallel)
    --entropy off Entropy-based secret detection
    --timeout-per-file 30.0 Per-file analysis timeout in seconds
